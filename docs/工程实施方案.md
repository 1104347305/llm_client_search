- 构建生产级AI客户搜索系统
- 支持日均10万+查询请求
- 保证系统高可用性（99.9%）
- 实现规则热更新和灰度发布

### 1.3 实施周期
- **总周期**: 6周
- **开发阶段**: 4周

### 2.3 数据流程设计

```
用户请求
    ↓
[1] API Gateway 认证鉴权
    ↓
[2] 限流检查 (每用户100 QPS)
    ↓
[3] 查询路由器接收请求
    ↓
[4] 逻辑词检测
    ├─ 有逻辑词 → 直接转 Level 4
    └─ 无逻辑词 → 继续
    ↓
[5] Level 1 规则引擎 (Jieba分词)
    ├─ 匹配成功 → 返回结果 (记录日志)
    └─ 未匹配 → 继续
    ↓
[6] Level 2 增强模板匹配 (正则+YAML)
    ├─ 匹配成功 → 返回结果 (记录日志)
    └─ 未匹配 → 继续
    ↓
[7] Level 3 语义缓存查询 (Redis)
    ├─ 命中缓存 → 返回结果 (记录日志)
    └─ 未命中 → 继续
    ↓
[8] Level 4 LLM解析 (Agno+DashScope)
    ├─ 解析成功 → 缓存结果 → 返回
    └─ 解析失败 → 返回错误
    ↓
[9] 调用客户数据服务 (Elasticsearch)
    ↓
[10] 返回查询结果
    ↓
[11] 记录请求日志到数据库
    ↓
[12] 更新监控指标 (Prometheus)
```

## 四、关键技术方案

### 4.1 规则热更新机制

#### 方案设计
```
配置中心 (Nacos/Etcd)
    ↓
配置变更通知
    ↓
服务监听配置变更
    ↓
重新加载规则配置
    ↓
原子替换规则引擎实例
    ↓
完成热更新 (无需重启)
```

#### 实现方案
```python
class ConfigWatcher:
    """配置监听器"""

    def __init__(self, config_center_url: str):
        self.config_center = ConfigCenter(config_center_url)
        self.matcher = Level2EnhancedMatcher()

    async def watch(self):
        """监听配置变更"""
        async for event in self.config_center.watch("enhanced_rules.yaml"):
            if event.type == "UPDATE":
                await self.reload_rules()

    async def reload_rules(self):
        """重新加载规则"""
        new_matcher = Level2EnhancedMatcher()
        # 原子替换
        self.matcher = new_matcher
        logger.info("Rules reloaded successfully")
```

### 4.2 灰度发布机制

#### 灰度策略
```
┌─────────────────────────────────────────┐
│  流量分配策略                            │
│  - 按agent_id哈希分流                   │
│  - 灰度比例: 5% → 20% → 50% → 100%     │
│  - 回滚时间: <5分钟                     │
└─────────────────────────────────────────┘
```

#### 实现方案
```python
class GrayReleaseRouter:
    """灰度发布路由器"""

    def __init__(self, gray_ratio: float = 0.0):
        self.gray_ratio = gray_ratio
        self.stable_version = "v1.0"
        self.gray_version = "v1.1"

    def route(self, agent_id: str) -> str:
        """根据agent_id路由到不同版本"""
        hash_value = hash(agent_id) % 100
        if hash_value < self.gray_ratio * 100:
            return self.gray_version
        return self.stable_version
```

### 4.3 限流熔断机制

#### 限流策略
| 维度 | 限流阈值 | 说明 |
|------|----------|------|
| 全局QPS | 3000 | 所有请求 |
| 单用户QPS | 100 | 按agent_id限流 |
| LLM调用QPS | 500 | Level 4限流 |

#### 熔断策略
```python
class CircuitBreaker:
    """熔断器"""

    def __init__(self,
                 failure_threshold: int = 5,
                 timeout: int = 60):
        self.failure_threshold = failure_threshold
        self.timeout = timeout
        self.failure_count = 0
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN

    async def call(self, func, *args, **kwargs):
        """调用受保护的函数"""
        if self.state == "OPEN":
            if time.time() - self.open_time > self.timeout:
                self.state = "HALF_OPEN"
            else:
                raise CircuitBreakerOpenError()

        try:
            result = await func(*args, **kwargs)
            if self.state == "HALF_OPEN":
                self.state = "CLOSED"
                self.failure_count = 0
            return result
        except Exception as e:
            self.failure_count += 1
            if self.failure_count >= self.failure_threshold:
                self.state = "OPEN"
                self.open_time = time.time()
            raise e
```

CREATE TABLE request_logs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    request_id VARCHAR(64) NOT NULL COMMENT '请求ID',
    agent_id VARCHAR(64) COMMENT '代理人ID',
    query TEXT NOT NULL COMMENT '查询文本',
    matched_level INT COMMENT '命中层级',
    confidence DECIMAL(3,2) COMMENT '置信度',
    conditions JSON COMMENT '查询条件',
    query_logic VARCHAR(10) COMMENT '查询逻辑',
    response_time INT COMMENT '响应时间(ms)',
    result_count INT COMMENT '结果数量',
    error_message TEXT COMMENT '错误信息',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_agent_id (agent_id),
    INDEX idx_matched_level (matched_level),
    INDEX idx_created_at (created_at)
