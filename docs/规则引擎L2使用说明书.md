## 1. 概述

L2 增强模板匹配器是系统中**覆盖最广、最可控**的解析层。它基于 YAML 配置文件定义规则，通过正则表达式匹配自然语言查询，输出结构化的 `Condition` 对象。

### 核心特性

- **纯配置驱动**：所有规则在 YAML 中定义，无需修改 Python 代码
- **枚举自动展开**：`{enum}` 占位符自动替换为字段的所有枚举值
- **别名归一化**：预处理阶段将口语别名替换为标准值
- **否定词支持**：自动处理"未/没有/不"等否定前缀
- **配对字段约束**：某些字段必须成对出现，否则丢弃
- **动态日期范围**：运行时计算"本月/下月/N天内"等相对日期
- **冲突解决**：同位置多规则命中时，按优先级取最高者

当前规则数量：**153 条**，覆盖字段 **70+**。

## 2. YAML 文件整体结构

```yaml
# ============================================================
# 规则列表
# ============================================================
rules:
  - name: "规则名称"
    # ... 规则字段

# ============================================================
# 枚举值（内联，适合小型枚举）
# ============================================================
enum_values:
  field_name:
    - "值1"
    - "值2"

# ============================================================
# 枚举文件（外部引用，适合大型枚举）
# ============================================================
enum_files:
  field_name: enums/field_name.yaml

# ============================================================
# 别名映射（别名 → 标准值）
# ============================================================
value_mappings:
  field_name:
    "别名1": "标准值"
    "别名2": "标准值"

# ============================================================
# 枚举排序（用于 enum_gte / enum_lte 展开）
# ============================================================
enum_orders:
  field_name:
    - "最低值"
    - "中间值"
    - "最高值"

# ============================================================
# 否定词列表
# ============================================================
negation_words:
  - "未"
  - "没有"
  - "不"
```

## 3. 规则字段详解

### 完整字段一览

```yaml
- name: "规则唯一名称"           # 必填，用于日志和调试
  patterns:                      # 必填（与 patterns_template 二选一）
    - '正则表达式1'
    - '正则表达式2'
  patterns_template:             # 与 patterns 二选一，含 {enum}/{negation} 占位符
    - '.*?{enum}.*?'
  enum_ref: "field_name"         # 配合 patterns_template，指定枚举来源字段
  field: "es_field_name"         # 必填，目标字段名（支持点号嵌套：policies.due_date）
  operator: "MATCH"              # 必填，操作符-MATCH, GTE, LTE, RANGE, CONTAINS, NOT_CONTAINS, NESTED_MATCH（见第5节）
  value_type: "capture"          # 必填，值提取方式-static、capture、range、enum_gte、enum_lte、date_range_dynamic、none（见第4节）
  value:                         # 值配置（根据 value_type 不同而不同）
    group: 1                     # capture 时：捕获组编号
    transform: "int"             # 可选转换-int、multiply、plus_range、exact_range、year_to_birth_range、chinese_decade_plus_range（见第6节）
    multiplier: 10000            # transform=multiply，时的乘数
  priority: 8                    # 优先级，越高越优先（冲突时使用）
  negation_support: true         # 是否支持否定词（默认 false）
  require_paired_field: "xxx"    # 配对字段约束（见第10节）
```

### 字段说明

#### `name`（必填）

规则的唯一标识符，用于日志输出和调试。建议格式：`"字段名-规则描述"`。

```yaml
name: "年龄-以上"
name: "寿险VIP-精确匹配"
name: "保单号-匹配"
```

#### `patterns`（必填之一）

手写正则表达式列表。每个 pattern 都会在查询文本上执行 `finditer`。

**写法要点：**
- 使用 `.*?` 非贪婪匹配前后文本（允许前后有其他内容）
- 用 `()` 捕获需要提取的值（配合 `value.group`）
- 用 `(?:...)` 作为非捕获分组
- 中文字符范围：`[\u4e00-\u9fa5]`

```yaml
patterns:
  - '.*?(\d+)岁以上.*?'           # 捕获年龄数字
  - '.*?年龄.*?超过(\d+).*?'
  - '.*?(\d+)岁?以上的客户.*?'
```

#### `patterns_template`（必填之一）

含 `{enum}` 或 `{negation}` 占位符的模板，自动展开为包含所有枚举值的 patterns。

- `{enum}` → 展开为 `(值1|值2|值3)` 捕获组
- `{negation}` → 展开为 `(?:未|没有|不|...)` 非捕获组

```yaml
patterns_template:
  - '.*?{enum}.*?'                 # 直接匹配任意枚举值
  - '.*?寿险VIP.*?{enum}.*?'      # 加上语境关键词
  - '{negation}{enum}'             # 否定词+枚举（用于 NOT_CONTAINS）
```

必须同时指定 `enum_ref`：

```yaml
enum_ref: "life_insurance_vip"    # 引用 enum_values 或 enum_files 中的字段
```

#### `field`（必填）

目标字段名，与后端搜索 API 的字段定义一致。嵌套字段用点号分隔：

```yaml
field: "age"                      # 顶层字段
field: "policies.policy_id"       # 嵌套字段（保单数组内的保单号）
field: "policies.due_date"        # 嵌套字段（保单缴费日期）
field: "family_members.age"       # 嵌套字段（家庭成员年龄）
```

#### `priority`（必填）

整数，值越大优先级越高。当不同规则匹配到文本中**同一位置**时，保留优先级最高的规则。

**推荐优先级参考：**

| 级别 | 值 | 适用场景 |
|------|----|---------|
| 极高 | 11-15 | 唯一标识符（保单号、证件号、姓名） |
| 高 | 8-10 | 枚举精确匹配、日期匹配 |
| 中 | 6-7 | 数值范围、模糊类型匹配 |
| 低 | 1-5 | 兜底规则 |

## 4. value_type 类型说明

### `static` — 静态固定值

规则命中即返回固定值，不从正则中提取。

```yaml
value_type: "static"
value: "是"           # 直接返回字符串
```

**适用场景**：布尔类字段（缴费期满、是否已领取生存金等）。

```yaml
- name: "缴费期满-已满"
  patterns:
    - '.*?缴费.*?期满.*?'
  field: "is_payment_matured"
  operator: "MATCH"
  value_type: "static"
  value: "是"
  priority: 6
```

---

### `capture` — 从正则捕获组提取

从指定捕获组提取字符串，可选做数值转换。

```yaml
value_type: "capture"
value:
  group: 1             # 捕获组编号（从 1 开始）
  transform: "int"     # 可选，转换方式（见第6节）
  multiplier: 10000    # transform=multiply 时的乘数
```

```yaml
- name: "年龄-以上"
  patterns:
    - '.*?(\d+)岁以上.*?'
  field: "age"
  operator: "GTE"
  value_type: "capture"
  value:
    group: 1
    transform: "int"    # 将 "45" 转为 45（整数）
  priority: 8
```

---

### `range` — 从两个捕获组构建范围

从两个捕获组分别提取最小值和最大值，构建 `RangeValue`。

```yaml
value_type: "range"
value:
  min_group: 1         # 最小值的捕获组
  max_group: 2         # 最大值的捕获组
  transform: "int"     # 可选，对两个值都应用转换
```

也支持**静态范围**（无捕获组）：

```yaml
value_type: "range"
value:
  min: 0
  max: 100
```

```yaml
- name: "年龄-区间"
  patterns:
    - '.*?(\d+)[-~到至](\d+)岁.*?'
  field: "age"
  operator: "RANGE"
  value_type: "range"
  value:
    min_group: 1
    max_group: 2
    transform: "int"
  priority: 9
```

---

### `enum_gte` / `enum_lte` — 枚举有序范围

用于枚举字段的"以上/以下"查询，返回枚举顺序中从该值起的所有值（列表）。

需要在 `enum_orders` 中定义字段的枚举排序。

```yaml
value_type: "enum_gte"   # 或 "enum_lte"
value:
  group: 1
```

运行时，`"黄金V1以上"` → 捕获 `"黄金V1"` → 查 `enum_orders.life_insurance_vip` → 返回 `["黄金V1", "黄金V2", "黄金V3", "原黄金VIP", "铂金V1", "原铂金VIP"]`，operator 自动变为 `CONTAINS`。

```yaml
- name: "寿险VIP-以上"
  enum_ref: "life_insurance_vip"
  patterns_template:
    - '.*?{enum}(?:以上|及以上).*?'
  field: "life_insurance_vip"
  operator: "CONTAINS"
  value_type: "enum_gte"
  value:
    group: 1
  priority: 9
```

---

### `date_range_dynamic` — 动态日期范围

运行时动态计算日期范围，无需硬编码日期。详见 [第9节](#9-动态日期范围)。

```yaml
value_type: "date_range_dynamic"
value:
  date_range: "next_month"    # 类型（见第9节）
  format: "YYYY-MM-DD"        # 日期格式
```

---

### `none` — 无值（EXISTS/NOT_EXISTS）

用于只判断字段是否存在，不关心具体值。

```yaml
value_type: "none"
operator: "EXISTS"       # 或 "NOT_EXISTS"
```

## 5. operator 操作符说明

| 操作符 | 说明 | value 类型 | 适用场景 |
|--------|------|------------|---------|
| `MATCH` | 精确匹配 | string | 枚举字段精确值、客户号、保单号 |
| `GTE` | 大于等于 | number | 年龄、收入等数值下限 |
| `LTE` | 小于等于 | number | 年龄、收入等数值上限 |
| `RANGE` | 区间范围 | RangeValue | 年龄区间、日期区间 |
| `CONTAINS` | 包含（列表中的任一值） | List[str] | 枚举以上/以下（展开为列表） |
| `NOT_CONTAINS` | 不包含 | List[str] / str | 未持有某险种 |
| `EXISTS` | 字段存在且不为空 | 无 | 判断是否有某类数据 |
| `NOT_EXISTS` | 字段不存在或为空 | 无 | 判断是否缺少某类数据 |
| `NESTED_MATCH` | 嵌套数组内精确匹配 | string | 保单、家庭成员、证件等嵌套字段 |

**注意**：`NESTED_MATCH` 用于嵌套数组字段（如 `policies.*`, `family_members.*`），表示在同一数组元素内匹配。YAML 中写 `NESTED_MATCH`，发送时自动转换为 `MATCH`，由后端识别点号字段名判断是否嵌套查询。

## 6. transform 转换说明

`transform` 在 `value_type: capture` 和 `value_type: range` 时可选。

| transform | 说明 | 参数 | 示例 |
|-----------|------|------|------|
| `int` | 转为整数 | — | `"45"` → `45` |
| `multiply` | 乘以倍数 | `multiplier` | `"10"万` → `100000` |
| `plus_range` | 基数+偏移构建范围 | `offset`, `range` | `"20"多岁` → `RangeValue(21,29)` |
| `exact_range` | 单值转为精确范围 | — | `"45"` → `RangeValue(45,45)` |
| `year_to_birth_range` | 出生年份转日期范围 | — | `"1953"` → `RangeValue("19530101","19531231")` |
| `chinese_decade_plus_range` | 中文年代词转范围 | `offset`, `range` | `"二十"多岁` → `RangeValue(21,29)` |

### 示例：万元转换

```yaml
- name: "年收入-以上"
  patterns:
    - '.*?年收入.*?(\d+)万.*?以上.*?'
    - '.*?年收入.*?超过(\d+)万.*?'
  field: "annual_income"
  operator: "GTE"
  value_type: "capture"
  value:
    group: 1
    transform: "multiply"
    multiplier: 10000        # 10万 → 100000
  priority: 8
```

### 示例：出生年份转日期范围

```yaml
- name: "出生年份-范围"
  patterns:
    - '.*?(\d{4})年(?:出生|出生的|生).*?'
  field: "client_birth"
  operator: "RANGE"
  value_type: "capture"
  value:
    group: 1
    transform: "year_to_birth_range"    # "1953" → {min:"19530101", max:"19531231"}
  priority: 8
```

## 7. 枚举展开机制

枚举展开是 L2 最核心的功能之一，允许用一条规则模板覆盖所有枚举值。

### 7.1 定义枚举值

**方式一：内联（小型枚举，直接在 YAML 中列出）**

```yaml
enum_values:
  marital_status:
    - "未婚"
    - "已婚"
    - "离异"
    - "丧偶"
    - "分居"
```

**方式二：外部文件（大型枚举，独立 YAML 文件）**

```yaml
enum_files:
  life_insurance_product: enums/life_insurance_product.yaml  # 相对于 config/
```

### 7.2 在规则中引用

```yaml
- name: "婚姻状况"
  enum_ref: "marital_status"       # 引用上面定义的枚举
  patterns_template:
    - '.*?{enum}.*?'               # {enum} 展开为 (未婚|已婚|离异|丧偶|分居)
  field: "marital_status"
  operator: "MATCH"
  value_type: "capture"
  value:
    group: 1
  priority: 10
```

展开后等效于：

```yaml
patterns:
  - '.*?(未婚|已婚|离异|丧偶|分居).*?'
```

### 7.3 枚举排序（用于以上/以下）

配合 `enum_gte` / `enum_lte` 使用，定义枚举的升序排列：

```yaml
enum_orders:
  life_insurance_vip:
    - "黄金V1"       # 最低
    - "黄金V2"
    - "黄金V3"
    - "原黄金VIP"
    - "铂金V1"
    - "原铂金VIP"   # 最高
    - "全部"
```

查询"黄金V2以上"时，返回：`["黄金V2", "黄金V3", "原黄金VIP", "铂金V1", "原铂金VIP", "全部"]`。

## 8. 别名归一化机制

`value_mappings` 在**预处理阶段**将查询文本中的别名替换为标准值，使得 pattern 只需匹配标准值。

### 8.1 定义别名

```yaml
value_mappings:
  education:
    "本科":   "大学本科"    # "本科" → "大学本科"
    "研究生": "硕士研究生"
    "博士":   "博士研究生"

  life_insurance_vip:
    "铂金":   "铂金V1"      # "铂金" → "铂金V1"
    "黄金":   "黄金V1"
    # 自映射（防止短串误匹配）
    "黄金V1": "黄金V1"
    "铂金V1": "铂金V1"
```

### 8.2 工作原理

预处理使用**单次扫描正则替换**（长串优先），避免级联替换：

```
查询："持有铂金VIP的客户"
  ↓ 预处理（铂金→铂金V1）
归一化后："持有铂金V1VIP的客户"   ← 仅替换"铂金"为"铂金V1"
```

**注意**：避免定义含有中文通用词（如"近"、"内"）的短别名，否则会误匹配：

```yaml
# ❌ 错误示例：会误匹配"最近一年内"中的"近一年"
latest_underwriting_time:
  "近一年": "一年内"    # 会把"最近一年内"变成"最一年内内"！

# ✅ 正确：只用无歧义的别名
latest_underwriting_time:
  "1年内": "一年内"     # 安全，不会误匹配其他词
```

## 9. 动态日期范围

用于需要在运行时计算相对日期的场景（本月、下月、N天内等）。

### 9.1 支持的 date_range 类型

| 类型 | 说明 | 示例结果（假设今天 2026-03-08） |
|------|------|-------------------------------|
| `next_month` | 下个月1日～月末 | `{min:"2026-04-01", max:"2026-04-30"}` |
| `current_month` | 本月1日～月末 | `{min:"2026-03-01", max:"2026-03-31"}` |
| `next_n_days` | 今天起N天内 | N=30: `{min:"2026-03-08", max:"2026-04-07"}` |
| `last_n_days` | 过去N天内 | N=30: `{min:"2026-02-06", max:"2026-03-08"}` |

### 9.2 日期格式

```yaml
format: "YYYY-MM-DD"    # → "2026-04-01"（默认）
format: "YYYYMMDD"      # → "20260401"
```

### 9.3 从捕获组读取天数

`next_n_days` / `last_n_days` 支持从正则捕获组动态读取天数：

```yaml
- name: "缴费日期-N天内"
  patterns:
    - '.*?(\d+)天内.*?(?:需要|应|要)?缴费.*?'
  field: "policies.due_date"
  operator: "RANGE"
  value_type: "date_range_dynamic"
  value:
    date_range: "next_n_days"
    days_group: 1          # 从捕获组1读取天数（如 "30天内" → days=30）
    format: "YYYY-MM-DD"
  priority: 9
```

### 9.4 完整示例

```yaml
- name: "缴费日期-下个月"
  patterns:
    - '.*?下个?月.*?(?:需要|应|要)?缴费.*?'
    - '.*?(?:需要|应|要)?缴费.*?下个?月.*?'
    - '.*?下个?月.*?(?:续费|续保|应缴).*?'
  field: "policies.due_date"
  operator: "RANGE"
  value_type: "date_range_dynamic"
  value:
    date_range: "next_month"
    format: "YYYY-MM-DD"
  priority: 9

- name: "缴费日期-本月"
  patterns:
    - '.*?本月.*?(?:需要|应|要)?缴费.*?'
    - '.*?当月.*?应缴.*?'
  field: "policies.due_date"
  operator: "RANGE"
  value_type: "date_range_dynamic"
  value:
    date_range: "current_month"
    format: "YYYY-MM-DD"
  priority: 9
```

## 10. 配对字段约束

某些嵌套字段在 API 语义上需要成对出现，否则单独使用没有意义或会产生错误结果。

### 配置方式

```yaml
- name: "证件类型"
  field: "certificates.id_type"
  # ...
  require_paired_field: "certificates.id_number"   # id_type 必须和 id_number 同时存在
```

### 运行逻辑

匹配完成后，若 `certificates.id_type` 出现在条件列表中但 `certificates.id_number` 不在，则**丢弃 `id_type` 条件**，查询降级到 L4 LLM 处理。

### 适用场景

| 字段 | 配对字段 | 原因 |
|------|---------|------|
| `certificates.id_type` | `certificates.id_number` | API 要求证件类型和号码成对查询 |

## 11. 冲突解决策略

当多条规则匹配到查询文本中**位置重叠的片段**时，按以下策略解决：

1. **同字段冲突**：保留优先级最高的一个
2. **位置重叠冲突**：保留优先级最高的一个
3. **不重叠、不同字段**：共存，同时生效

**示例**：

查询 `"保单号P644037的客户"`：

| 规则 | 匹配位置 | 优先级 |
|------|---------|--------|
| `保单号-匹配` | "P644037" | 10 |
| `客户号-匹配` | 无（P前缀已被排除） | 8 |

→ 最终：`policies.policy_id MATCH "P644037"`

## 12. 完整示例

以下展示各种规则类型的完整写法。

### 示例 1：数值范围（年龄区间）

```yaml
- name: "年龄-区间"
  patterns:
    - '.*?(\d+)[-~到至](\d+)岁.*?'
    - '.*?(\d+)岁.*?到.*?(\d+)岁.*?'
  field: "age"
  operator: "RANGE"
  value_type: "range"
  value:
    min_group: 1
    max_group: 2
    transform: "int"
  priority: 9
```

### 示例 2：枚举精确匹配（婚姻状况）

```yaml
# 枚举值定义
enum_values:
  marital_status:
    - "未婚"
    - "已婚"
    - "离异"
    - "丧偶"
    - "分居"

# 别名
value_mappings:
  marital_status:
    "单身": "未婚"
    "结婚": "已婚"
    "结婚了": "已婚"

# 规则
- name: "婚姻状况"
  enum_ref: "marital_status"
  patterns_template:
    - '.*?{enum}.*?'
  field: "marital_status"
  operator: "MATCH"
  value_type: "capture"
  value:
    group: 1
  priority: 10
```

### 示例 3：枚举以上（寿险VIP等级）

```yaml
# 枚举排序（必须按低→高排列）
enum_orders:
  life_insurance_vip:
    - "黄金V1"
    - "黄金V2"
    - "黄金V3"
    - "原黄金VIP"
    - "铂金V1"
    - "原铂金VIP"
    - "全部"

# 规则（精确匹配）
- name: "寿险VIP"
  enum_ref: "life_insurance_vip"
  patterns_template:
    - '.*?寿险VIP.*?{enum}(?!.*(?:以上|及以上|以下|及以下)).*?'
  field: "life_insurance_vip"
  operator: "MATCH"
  value_type: "capture"
  value:
    group: 1
  priority: 8

# 规则（以上）
- name: "寿险VIP-以上"
  enum_ref: "life_insurance_vip"
  patterns_template:
    - '.*?{enum}(?:以上|及以上).*?'
  field: "life_insurance_vip"
  operator: "CONTAINS"
  value_type: "enum_gte"
  value:
    group: 1
  priority: 9
```

### 示例 4：否定词支持（险种未配置）

```yaml
- name: "险种-未配置"
  enum_ref: "life_liability_type"
  patterns_template:
    - '{negation}.*?{enum}.*?'
    - '.*?未配置.*?{enum}.*?'
    - '.*?没有.*?{enum}.*?'
  field: "life_liability_type"
  operator: "NOT_CONTAINS"
  value_type: "capture"
  value:
    group: 1
  priority: 8
  negation_support: true
```

### 示例 5：嵌套字段匹配（保单号）

```yaml
- name: "保单号-匹配"
  patterns:
    - '.*?(P\d{6,15}).*?'
    - '.*?保单号.*?[为是：:]\s*(P?\d{1,15}).*?'
    - '.*?保单号.*?(P?\d{6,15})(?:的客户|客户)?.*?'
  field: "policies.policy_id"
  operator: "NESTED_MATCH"
  value_type: "capture"
  value:
    group: 1
  priority: 10
```

### 示例 6：动态日期（下个月缴费）

```yaml
- name: "缴费日期-下个月"
  patterns:
    - '.*?下个?月.*?(?:需要|应|要)?缴费.*?'
    - '.*?下个?月.*?(?:续费|续保|应缴).*?'
  field: "policies.due_date"
  operator: "RANGE"
  value_type: "date_range_dynamic"
  value:
    date_range: "next_month"
    format: "YYYY-MM-DD"
  priority: 9
```

### 示例 7：出生年份（year_to_birth_range）

```yaml
- name: "出生年份-范围"
  patterns:
    - '.*?(\d{4})年(?:出生|出生的|生).*?'
    - '.*?出生于(\d{4})年.*?'
  field: "client_birth"
  operator: "RANGE"
  value_type: "capture"
  value:
    group: 1
    transform: "year_to_birth_range"    # "1953" → {min:"19530101",max:"19531231"}
  priority: 8
```

### 方法三：批量回归测试

```python
import asyncio
from app.core.query_router import QueryRouter

async def regression():
    router = QueryRouter()
    cases = [
        ("下个月需要缴费的客户",    2, "policies.due_date"),
        ("1953年出生的客户",         2, "client_birth"),
```

## 14. 常见错误

### 规则不生效

**现象**：某查询应被 L2 匹配，但实际走了 L4。

**排查步骤**：

1. 用调试方法二测试，看归一化后的文本是否还包含目标词
2. 检查正则是否正确（Python re 语法）
3. 检查 priority 是否被更高优先级规则覆盖
4. 检查 `require_paired_field` 是否导致条件被丢弃

### 误匹配（错误字段被捕获）

**现象**：查询条件解析到了错误字段，如数字被误识别为客户号。

**排查**：通常是通配性 pattern（如 `.*?(\d+).*?`）优先级过高覆盖了正确规则。

**解决**：
- 提高正确规则的 priority
- 收窄错误规则的 pattern（加前缀/后缀约束）

### 别名替换异常

**现象**：归一化后文本出现重复或截断，如"最近一年内内"。

**原因**：别名与常用词产生部分重叠（如"近一年"匹配了"最近"中的"近"）。

**解决**：去掉包含常用词的短别名，仅保留无歧义的别名。

### 枚举展开失败

**现象**：日志显示 `Rule 'xxx' has enum_ref='yyy' but no enum_values defined`。

**解决**：检查 `enum_values` 或 `enum_files` 中是否存在对应的 key。

### RANGE 值被 Pydantic 拒绝

**现象**：日期字符串（如 `"2026-04-01"`）被 `RangeValue` 拒绝，报类型错误。

**解决**：确认 `RangeValue` 的类型声明包含 `str`：

```python
class RangeValue(BaseModel):
    min: Optional[Union[int, float, str]] = None  # 必须包含 str
    max: Optional[Union[int, float, str]] = None
```
