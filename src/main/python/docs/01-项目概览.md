# 01-项目概览

## 项目定位

`llm_client_search` 是一个自然语言客户搜索条件解析服务。它接收中文自然语言查询，例如：

```text
45岁以上且买过重疾险的客户
```

然后输出结构化查询条件：

```json
{
  "query_logic": "AND",
  "conditions": [
    {
      "field": "clientBirthday",
      "operator": "LTE",
      "value": "1981-12-31 00:00:00"
    }
  ]
}
```

当前项目主能力是“解析不搜索”。代码里有调用下游客户搜索 API 的 `SearchService`，但当前 FastAPI router 暴露的主接口是解析、字段检索和配置热更新。

## 技术栈

| 类型 | 技术 |
| --- | --- |
| Web 框架 | FastAPI |
| 数据模型 | Pydantic v2 |
| LLM 客户端 | OpenAI 兼容接口，当前配置指向 DashScope |
| HTTP 客户端 | httpx |
| 中文分词 | jieba |
| 配置格式 | YAML |
| 字段 RAG | Elasticsearch BM25 + Trie 枚举召回 + L2 候选召回 |
| 日志 | loguru |
| 缓存 | Redis 模块存在，但主链路 L3 当前未启用 |

## 核心能力

- L1：手机号、身份证、客户号、保单号、姓名等确定性实体提取。
- L2：基于 YAML 的增强规则匹配，当前 417 条规则、70 条组合规则。
- L4：LLM 兜底解析，结合字段意图 RAG，避免模型凭空生成字段。
- 字段 schema 集中配置，当前 `field_definitions_args.yaml` 有 179 条字段意图。
- 支持配置热更新、按文件热更新、字段意图 ES 索引重建。
- 支持 AskBob 标准协议响应。
- 支持响应字段 AES 加密开关。
- 支持日志敏感信息脱敏。

## 运行方式

安装依赖：

```bash
cd /Users/mickey/project/PA-ALG/llm_client_search
pip install -r src/main/python/requirements.txt
```

启动服务：

```bash
ENV=dev python -m src.main.python.main
```

默认端口来自 `src/main/python/config/dev_client_search_args.yaml`：

```yaml
API_HOST: "0.0.0.0"
API_PORT: 8000
```

访问：

```text
http://localhost:8000/health
http://localhost:8000/docs
```

## 环境选择

配置加载由 `ENV` 决定：

```text
ENV=dev -> config/dev_client_search_args.yaml
ENV=stg -> config/stg_client_search_args.yaml
ENV=prd -> config/prd_client_search_args.yaml
```

加载优先级：

```text
环境变量 > 当前 ENV 对应 YAML > 代码默认值
```
