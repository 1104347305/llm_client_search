## FastAPI 启动流程

入口：`src/main/python/main.py`

主要步骤：

1. 设置 `PACKAGE_ROOT` 和 `PROJECT_ROOT` 到 `sys.path`。
2. 创建 FastAPI app。
3. 注册 CORS。
4. 注册 router：

   ```python
   app.include_router(router, prefix="/api/v1", tags=["search"])
   ```

5. startup 时后台预热 `QueryRouter`。
6. 启动 runtime reload marker watcher。
7. 暴露 `/` 和 `/health`。

## 解析主链路

核心入口：

```python
await QueryRouter().route_with_peeling(query)
```

调用链：

```text
api/client_search_query_parse_post.py
  -> get_query_router()
  -> QueryRouter.route_with_peeling()
  -> Level1RuleEngine.extract()
  -> Level2EnhancedMatcher.match()
  -> Level4LLMParser.parse()
  -> intent_summary.build_intent_summary()
```

主流程：

```text
原始 query
  -> 去空格
  -> FieldRegistry.normalize_query()
  -> 记录裸姓名/裸值候选
  -> L1 确定性实体提取
  -> L2 增强规则匹配
  -> 如果 L2 命中，丢弃 L1 条件
  -> L3 当前跳过
  -> 无条件时进入 L4
  -> 字段合法性校验
  -> 条件结构归一
  -> 意图摘要
  -> AskBob 协议响应
```

## L1：确定性实体

文件：`steps/level1_rule_engine.py`

支持：

| 类型 | 规则 |
| --- | --- |
| 客户号 | `C` + 15 位数字 |
| 保单号 | `P` + 15 位数字 |
| 身份证号 | 18 位，末位可为 X/x |
| 手机号 | 11 位大陆手机号 |
| 人名 | jieba 词性 `nr` / `nrt` / `nrfg` |

输出字段不直接硬编码，而是通过 `get_query_field()` 从 `field_mapping_args.yaml` 读取。

## L2：YAML 增强规则

文件：`steps/level2_enhanced_matcher.py`

配置：`config/enhanced_rules_args.yaml`

当前规模：

```text
rules: 417
composite_rules: 70
pattern_vars: 2
```

能力：

- 正则规则匹配。
- `patterns_template` 模板展开。
- `{enum}` / `{multi_enum}` 枚举展开。
- 否定词与持有词模板。
- 多枚举值拆分。
- 口语值归一化。
- 组合规则引用。
- 弱命中裸值扩展。
- 给 L4 RAG 提供候选字段召回。

## L3：语义缓存

文件存在：`steps/level3_semantic_cache.py`

配置存在：

```yaml
ENABLE_L3: true
REDIS_HOST: localhost
REDIS_PORT: 6379
CACHE_TTL: 3600
SEMANTIC_CACHE_THRESHOLD: 0.85
```

但 `QueryRouter.route_with_peeling()` 中 L3 调用当前被注释，主链路没有实际使用 Redis 缓存。

## L4：LLM + RAG

文件：`steps/level4_llm_parser.py`

LLM 客户端：

```python
AsyncOpenAI(
    api_key=settings.LLM_API_KEY,
    base_url=settings.LLM_BASE_URL,
)
```

RAG 三路召回：

| 召回方式 | 作用 |
| --- | --- |
| ES BM25 | 用自然语言匹配字段意图 |
| Trie 枚举 | 用户 query 直接出现枚举值时精准召回 |
| L2 候选召回 | 复用 L2 规则片段召回字段 |

防幻觉策略：

- L4 只把 RAG 召回的字段 schema 注入 prompt。
- 如果 RAG 无召回，L4 返回空条件，不让模型自由编字段。
- L4 输出后还会经过 `QueryRouter._validate_conditions()` 校验字段合法性。

## 条件后处理

主要在 `QueryRouter` 中完成：

- 显式客户全名强制保留完整姓名。
- 裸姓名候选仅在无条件时升级为姓名条件。
- 删除不支持的角色字段。
- 删除未映射的空 EXISTS 条件。
- 非法字段整体返回空。
- `CONTAINS` 单值归一。
- `GTE + LTE` 合并成 `RANGE`。
- 家庭成员年龄可转换为生日范围。
