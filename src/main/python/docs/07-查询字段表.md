# 查询字段表

更新时间：2026-05-13

数据来源：`src/main/python/config/field_definitions_args.yaml` 与 `src/main/python/config/*_enums_args.yaml`。

当前字段 intent 数：179；支持 intent 数：157；不支持 intent 数：22；去重字段数：80。

字段统计：支持字段 68 个，不支持字段 12 个，部分支持字段 0 个。

说明：

- 支持状态根据 intent 的 `is_supported` 或 `is_support` 判断；只有显式为 `false` 时记为不支持，缺省按支持处理。
- 支持字段和不支持字段已拆分成独立表格，不混排。
- “支持操作查询”按字段数据类型推导：字符串/枚举类支持 `MATCH`、`CONTAINS`、`NOT_CONTAINS`、`EXISTS`、`NOT_EXISTS`；时间/数值类支持 `GTE`、`GT`、`LTE`、`LT`、`RANGE`。
- “已支持查询操作”根据当前配置文件中支持 intent 的 operator 生成。
- `show_enum_in_prompt: false` 的大枚举字段也会按 `enum_candidate_limit_in_prompt` 展示候选枚举，而不是留空。
- 字段格式来自 `value_type`、`unit`、`format`。
- 字段枚举值优先读取 intent 内的 `enum`，其次读取 `enum_ref` 指向的枚举配置。

## 操作符说明

| 操作符 | 含义 |
| --- | --- |
| `MATCH` | 匹配/精确匹配 |
| `CONTAINS` | 包含，通常 value 为列表 |
| `NOT_CONTAINS` | 不包含，通常 value 为列表 |
| `GT` | 大于 |
| `GTE` | 大于等于 |
| `LT` | 小于 |
| `LTE` | 小于等于 |
| `RANGE` | 范围，value 为 {min,max} |
| `EXISTS` | 字段存在且不为空 |
| `NOT_EXISTS` | 字段不存在或为空 |
